# License Strategy

If you want credit, public visibility, and room to build more later, the
strongest setup is **not** "just make everything open with one generic license."

Use a layered model.

## Recommended Licensing Model

### 1. Code

License the code under:

- **Apache-2.0**

Why:

- it is respected and standard
- it preserves copyright and notices
- it supports open-source credibility
- it gives you strong attribution presence in the codebase

### 2. Docs, Course Writing, and Teaching Guides

License the docs under:

- **CC BY-NC 4.0**

Why:

- you still get attribution
- people can share and adapt with credit
- commercial reuse is restricted unless you approve it

This is a better fit for course writing than a pure software license.

### 3. Brand, Story World, and Course Identity

Do **not** open-license your brand names by accident.

Reserve:

- `ADD`
- `Attention Driven Design`
- `ADD Open Courses`
- `Attention First`
- `ADD P2P`
- `Command Nexus`
- course branding
- logos
- special story-world names

with a trademark / brand-reservation note.

That lets people reuse the teaching structure while making it clear they do not
automatically own your course identity.

## What This Means In Practice

Public:

- code templates
- starter packs
- reusable docs

Reserved / controlled:

- ADD brand identity
- book and framework naming
- course brand
- story universe naming
- institution-specific identity

## Best-Fit Recommendation

For your goals, the best setup is:

- **public repo**
- **Apache-2.0 for code**
- **CC BY-NC 4.0 for docs**
- **brand/trademark reserved**

That gives you:

- visible authorship
- reuse with credit
- room to build a public reputation
- protection against people casually cloning the whole course identity
